◎ 项目简介

PolkaInk 以交易 calldata 为存储媒介，将 Polkadot 生态历史文档（Markdown）直接写入区块——无数据库、无云服务、无单点失败。内容通过质押成员制 + 社区投票达成共识后上链，由 Archive Council 守护内容伦理底线，并以三种链上 Soulbound NFT 激励真实贡献者。

历史一旦写入，任何全节点皆可独立验证，任何权力皆无法将其抹去。

六条核心原则

设计哲学：权力不应被购买，只能被赢得——贡献即信任，时间即证明

⚖️ 质押入场，贡献加权

88 PAS 是参与资格，不是话语权筹码。投票权重取决于写了多少被接受的内容、锁仓了多长时间。

📖 线性历史，不可篡改

提案是 Pull Request，投票通过即合并，内容永久写入链上 calldata。没有撤销，只有继续书写。

🚫 单人不可通过

权重硬上限 cap=2.00=T。单人最大实际权重 1.80，任何人单独投 YES 永远不足以通过提案。

🌐 贡献对等，平权基底

写作贡献 B_hist 与锁仓贡献 B_lock 上限相同，均为 +0.40。无哪种贡献设计成碾压另一种。

🛡️ 集体守护，透明否决

Archive Council 只在必要时踩刹车，每次否决必须附上链上理由，永久可查。

💰 激励参与，不扭曲投票

投票奖励通过 30 天 Epoch 发放，投 YES 或 NO 均可获奖，奖励与立场解耦，保持判断独立性。

🪪 Proof of Personhood Polkadot 原生

当前防女巫机制：88 PAS 质押门槛 + 单人权重上限 1.80 < T。未来方向：接入 Polkadot Proof of Personhood（PoP），以链上唯一真人身份替代或补充质押准入，实现更公平的一人一基础权重，从根本上消除多账号操控风险。

治理机制 — 投票权重 · 提案流程

投票权重公式

weight = min(1.0 + boost, 2.00) ← 硬上限 cap boost = B_hist + B_lock B_hist = 0.40 × ln(1 + creator_nft) / ln(21) 上限 +0.40（≈ 20 枚 Creator NFT） B_lock = 0.40 × ln(1 + lock_months) / ln(25) 3mo: +0.1723 · 12mo: +0.3187 · 24mo: +0.4000 Guardian NFT：无权重加成 单人最大实际 weight = 1.80 < T = 2.0 ←永远无法单独通过

提案通过三重门槛

① score > 2.0

score = Σ(choice × weight)，YES = +weight，NO = −weight，ABSTAIN = 0

② 参与率 ≥ 5%

= 实际参与总权重 / 提案创建时全网基础权重快照，分母不可操纵

③ Archive Council 未完成 5/7 veto

24h 审查窗口内，达到 5/7 集体否决则进入 CouncilVetoed 状态，触发 72h 冷却期

设计反思 — v1 → v2 → v3.4 的兴衰与重建

每一次版本迭代，都是对过度设计的一次修剪

v1.0x 2026-02 · 初稿

过度工程化的起点

引入 Archive Council 7 人委员会，但成员管理权在 Admin 手中，任何时候可替换，理论上的"去中心化守护"是伪命题。

DOT 持币 + NFT 双轨权重，OG Gold / Silver / Bronze 三级特权 NFT 由团队空投，实质上给了创始团队永久话语权优势。

问题：60% 赞成门槛 + 无质押门槛 + Subsquid 依赖 + IPFS 前端——复杂度极高，单点失败多，去中心化是口号。

v2.0x 2026-03 · 激进简化

矫枉过正的震荡

彻底移除 Archive Council，改为 OG Gold 单人 veto——"去精英化"的初衷，反而制造了更大的单点权力：一人 NO 就能杀死任何通过的提案。

引入 ReportManager 举报机制，初衷是社区自治内容争议。但举报可被武器化（批量建号举报）、与 calldata 永久存储的核心价值矛盾，且与 Council veto 职责重叠。

问题：6 种 NFT（OGBronze/Silver/Gold/Member/Creator/Author）体系膨胀，治理复杂度不降反升。没有 Council，谁来处理真正的极端内容风险？

v3.4 当前 · 有机平衡

克制与均衡的答案

重新引入 Archive Council，但彻底剥夺 Admin 对成员的控制权——7 人地址写入构造函数，移除 setMember()，Phase 1 前无人可替换。

废除 OG NFT 和 Author NFT，精简为 3 种 NFT；废除举报机制（calldata 永久存在，举报改变不了历史，只有 Council 集体 veto 和社区重新写版本才是正确的争议处理路径）。

收获：零特权管理员；单人最大实际权重 1.80 ＜ T=2.0 的铁律；Epoch 奖励与立场解耦——这不是取巧，是系统成熟的标志。

"每次反思的核心问题不是'功能够不够多'，而是'权力能否被滥用'。
v1 的权力集中在 Admin，v2 的权力集中在 OG Gold，v3.4 的权力分散在数学和合约里。"
