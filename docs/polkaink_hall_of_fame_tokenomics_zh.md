## 1. 产品定位与核心逻辑

### 1.1 是什么

**PolkaInk 年度名人堂**（以下简称"名人堂"）是每年举办一届、完全链上运行的 Polkadot 生态影响力人物选举。每届从全球社区提名的候选人中，选出 **10 位年度最具影响力人物**，永久写入 PolkaInk 链上历史档案。

它不是一个投票工具，而是一个**围绕注意力经济构建的完整链上金融产品**：

```
社区参与产生注意力
     ↓
注意力转化为链上经济行为（提名 / 投票 / 传播 / 下注）
     ↓
经济行为产生手续费与奖池
     ↓
手续费的一部分反哺 PolkaInk 核心协议
     ↓
核心协议增强权威性，进一步吸引下一届参与
```

### 1.2 为什么有人参与

| 角色 | 参与动机 |
|------|---------|
| **候选人** | 获得链上永久荣誉记录；名人堂 NFT 是 Web3 最高公信力背书 |
| **支持者（粉丝）** | 帮助偶像上榜；投票获得对应荣誉；押注正确有收益 |
| **传播者** | 转发链接获得传播奖励；无需质押，零门槛参与 |
| **投机者** | 预测市场提供纯收益机会，与候选人是否"好人"无关 |
| **社区观察者** | 名人堂记录成为 Polkadot 生态的权威年度史料 |

### 1.3 与 PolkaInk 主协议的关系

名人堂是 PolkaInk 主协议的**独立子模块**，共享链上基础设施（合约、Treasury、NFT 体系），但拥有独立的资金流和参数体系：

- 当选人物的贡献记录和入堂宣言，自动通过 PolkaInk 主协议的 VersionUpdate 流程写入链上档案
- 名人堂产生的协议收入，按比例注入 PolkaInk 主 Treasury
- 名人堂投票使用 PolkaInk 主协议的 Historian NFT 加成体系，两套系统激励互通

## 2. 年度赛历与选战阶段

每届名人堂选举历时约 **10 周**，分为 5 个阶段。每年 **10 月 1 日**启动，**12 月 15 日**颁奖，与 Polkadot 生态年终总结节点对齐。

```
第 1-2 周：提名期（Nomination Phase）
第 3-4 周：贡献公示期（Evidence Phase）
第 5-6 周：初选投票（Primary Vote）—— 淘汰至 30 人
第 7-8 周：决选投票（Final Vote）—— 选出 10 人
第 9 周：  公示与争议期（Challenge Period）
第 10 周： 颁奖与链上存档（Award & Archive）
```

### 2.1 阶段时间轴

| 阶段 | 时间 | 核心事件 | 开放的经济行为 |
|------|------|---------|--------------|
| 提名期 | W1–W2（14 天） | 提名开放，候选人池形成 | 提名费缴纳、传播挖矿开始 |
| 贡献公示期 | W3–W4（14 天） | 候选人提交贡献证据 | 证据质押、预测市场开盘 |
| 初选 | W5–W6（14 天） | 全成员投票，候选人从 N→30 | 投票、追加传播奖励、预测市场活跃 |
| 决选 | W7–W8（14 天） | 决选轮，30→10 | 投票权重加倍结算、预测市场封盘 |
| 公示争议期 | W9（7 天） | Archive Council 审查，可提出异议 | 争议保证金缴纳 |
| 颁奖存档 | W10（7 天） | 链上颁奖、NFT 铸造、收益结算 | 全部收益分发 |

### 2.2 多轮选战设计的经济意义

两轮投票（初选 + 决选）并非多余的复杂度，而是**两倍的经济密度**：

- 每轮投票需要单独质押 → 两轮 = 两次质押费收入
- 初选结果产生「悬念」→ 悬念推动预测市场二次活跃
- 决选前传播挖矿重置，传播者有动力再次传播 → 两波流量

## 3. 候选人提名机制

### 3.1 谁可以被提名

候选人须满足以下**任意一项**链上可验证条件（降低门槛，鼓励广泛参与）：

| 条件类别 | 具体要求 |
|---------|---------|
| 技术贡献 | Polkadot/Substrate GitHub 有效 PR 合并记录（链下核实后链上存证） |
| 治理贡献 | 参与过 Polkadot OpenGov 投票 ≥ 10 次，或发起过通过的国库提案 |
| 生态建设 | 在 Polkadot 生态创立项目，且项目有链上活动记录 |
| 社区影响 | 持有 PolkaInk Historian NFT ≥ 3 枚（即在 PolkaInk 有贡献记录） |
| 跨链知名度 | 被 ≥ 50 名不同地址的活跃 Polkadot 成员联署提名 |

> 候选人可以自我提名，也可以由他人代为提名。

### 3.2 提名费用结构

提名不是免费的，费用设计兼顾「过滤垃圾提名」与「不阻碍真实候选人」两个目标。

| 提名类型 | 费用 | 退款条件 | 不退款条件 |
|---------|------|---------|----------|
| **自我提名** | 50 DOT | 进入决选 Top 30 退还 50% | 初选即淘汰，全部进入奖池 |
| **社区代提名** | 20 DOT / 提名人 | 被提名人进入 Top 30，提名人退还 100% | 被提名人初选淘汰，全部进入奖池 |
| **联署提名**（≥50 人） | 0.5 DOT / 联署人 | 被提名人进入决选，联署人获得 1.2× 返还 | 被提名人初选淘汰，全部进入奖池 |

**提名费去向：**

```
提名费总收入
  ├── 60% → 名人堂奖池（最终分配给当选者支持者）
  ├── 25% → PolkaInk 主 Treasury（协议反哺）
  └── 15% → 传播激励池（激励社区扩散）
```

### 3.3 提名质量保证金

候选人（或其代理人）在提交提名时，须额外缴纳 **30 DOT 质量保证金**，与提名费分开记录。

- 若候选人进入初选并完整提交贡献证据 → 全额退还
- 若候选人进入初选但未按时提交贡献证据 → 扣罚 100%，进入奖池
- 此设计防止「幽灵候选人」占位后不参与

### 3.4 提名上限与防刷机制

| 规则 | 数值 | 原因 |
|------|------|------|
| 每届最大提名总数 | 200 人 | 防止候选人池过大导致注意力稀释 |
| 单地址最多代提名 | 3 人 | 防止单一势力批量操控候选池 |
| 同一候选人被提名上限 | 5 次 | 多于 5 次的提名费不退款直接进奖池 |
| 提名期结束后 | 候选池锁定 | 不可添加新候选人，防止后期注水 |

## 4. 贡献提交与量化

贡献证据是选举的核心内容，也是名人堂「历史价值」的来源。贡献不经量化就是空头支票；量化不透明就是黑箱操作。v3 选择**链上存证 + 多维评分 + 社区审核**三层机制。

### 4.1 贡献分类与链上存证方式

| 贡献类别 | 链上存证方式 | 基础分值范围 |
|---------|-----------|------------|
| **代码贡献** | GitHub PR 哈希 + 合并时间戳，通过预言机写入链上 | 10–100 分 / 项 |
| **治理参与** | OpenGov 投票记录链上可直接验证 | 5–30 分 / 次 |
| **国库提案** | 提案 ID + 通过状态链上可验证 | 50–500 分 / 项 |
| **生态项目** | 项目合约地址 + TVL / 用户数快照 | 20–300 分 / 项 |
| **PolkaInk 文档** | Historian NFT 数量 × 文档字数权重 | 15–200 分 / 项 |
| **社区影响力** | 传播系数（见第 6 节）在提名期的累计值 | 0–150 分 |
| **外部认可** | 媒体报道链接（IPFS 存储）+ 社区核实 | 0–100 分 |

### 4.2 综合影响力指数（CII）

每位候选人最终得到一个**综合影响力指数（Comprehensive Impact Index，CII）**，用于初选排名参考（非直接决定当选，但影响投票者判断）：

```
CII = W_code × 代码贡献分
    + W_gov  × 治理贡献分
    + W_eco  × 生态建设分
    + W_doc  × 文档贡献分
    + W_comm × 社区影响分

权重默认值（可由 DAO 每届调整）：
  W_code = 0.30
  W_gov  = 0.20
  W_eco  = 0.25
  W_doc  = 0.15
  W_comm = 0.10
```

> **注意：** CII 是参考信息，不是选票。最终当选由投票决定。CII 在候选人页面公开展示，帮助投票者做决策。

### 4.3 贡献提交费与激励

| 操作 | 费用 | 说明 |
|------|------|------|
| 提交贡献证据条目 | 1 DOT / 条 | 防止垃圾证据刷分 |
| 社区核实（其他成员验证证据） | 0 DOT | 鼓励参与，核实准确者获奖励 |
| 证据被核实为虚假 | 扣罚提交者 5 DOT + 该条目无效 | 防造假 |
| 成功核实虚假证据 | 举报者获得 3 DOT 奖励（来自被扣罚的 5 DOT） | 激励监督 |

**贡献提交费去向：**
```
贡献提交费
  ├── 40% → 核实激励池（分给参与核实的成员）
  ├── 35% → PolkaInk 主 Treasury
  └── 25% → 名人堂奖池
```

## 5. 投票机制

### 5.1 投票资格

参与名人堂投票无需额外质押，但权重受 PolkaInk 主协议成员状态影响：

| 投票人类型 | 基础票权 | 加成来源 |
|-----------|---------|---------|
| PolkaInk 活跃成员（已质押 88 DOT） | 1 票 | Historian NFT + 锁仓加成（继承主协议权重） |
| 非 PolkaInk 成员（持有 ≥ 10 DOT） | 0.3 票 | 无加成 |
| 持有传播 NFT（见第 6 节） | +0.1 票 / 枚，上限 +0.5 | 传播贡献额外激励 |

> 设计意图：允许非主协议成员以较低权重参与，扩大名人堂影响力；同时保证主协议忠实成员的话语权优势。

### 5.2 投票费用（选战燃料费）

每次投票需缴纳少量费用，这是名人堂最稳定的收入来源：

| 阶段 | 费用 / 票 | 说明 |
|------|---------|------|
| 初选投票 | 0.5 DOT | 每地址每位候选人 |
| 决选投票 | 1.0 DOT | 决选注意力更集中，溢价合理 |
| 追加支持（Super Vote） | 5 DOT | 可额外增加 3× 权重，但每地址限 1 次 / 届 |

**投票费去向：**
```
投票费总收入
  ├── 50% → 名人堂奖池（支持获胜候选人的投票者分成）
  ├── 30% → PolkaInk 主 Treasury（协议反哺）
  └── 20% → 传播激励池
```

### 5.3 投票结果计算

#### 初选（Top 30 选拔）

```
初选得票 = Σ(投票权重_i × 投票数_i)

所有候选人按初选得票降序排列
前 30 名进入决选
第 31 名及以后淘汰
```

#### 决选（Top 10 产生）

```
决选有效票 = 初选得票 × 0.3 + 决选新增得票 × 0.7

（保留初选基础，但决选期间的新增投票权重更大，
 给后来居上者机会，保持悬念）

前 10 名入选年度名人堂
```

### 5.4 投票者收益分配

投票给最终入选 Top 10 候选人的投票者，可按比例分享奖池：

```
单个投票者分成 =（该投票者对该候选人的总投票权重 /
                 该候选人所有投票者的总权重）
               × 该候选人对应的奖池份额

候选人奖池份额按最终得票排名线性分配：
  第 1 名 → 20% 奖池
  第 2 名 → 16%
  第 3 名 → 13%
  第 4 名 → 10%
  第 5 名 → 9%
  第 6–10 名 → 各 6.4%（共 32%）
```

## 6. 社区传播激励（转发即挖矿）

这是名人堂区别于传统投票产品的核心创新：**让社区传播行为直接产生链上经济收益**，将 Web2 流量转化为 Web3 价值。

### 6.1 传播链接生成机制

任何持有 PolkaInk 账户的用户（无需是正式成员）都可以：

1. 在名人堂页面为任意候选人生成**个人专属推广链接**（链上记录推广人地址）
2. 将链接分享到社交媒体（Twitter/X、Telegram、Discord 等）
3. 通过该链接完成的**每笔经济行为**（投票、下注、提名），推广人获得对应的传播奖励

### 6.2 传播奖励计算

```
传播奖励 = 被推广行为金额 × 传播费率

传播费率：
  下游投票费    → 推广人获 8%
  下游预测下注  → 推广人获 5%
  下游提名费    → 推广人获 3%

传播奖励上限（防止超额奖励）：
  单届单地址最高传播收益 = 500 DOT
```

**示例：** 用户 A 分享候选人 X 的推广链接 → 100 人通过该链接各投票 1 次（每次 0.5 DOT）→ 用户 A 获得 `100 × 0.5 × 8% = 4 DOT` 传播奖励。

### 6.3 二级传播（裂变奖励）

支持最多 **2 级裂变**，鼓励传播者主动招募传播者：

```
一级传播者（直接推广链接产生的行为）：
  费率 100%（即上述费率）

二级传播者（通过一级传播者的链接注册后，
           二级传播者自己的传播行为）：
  二级传播者获得 70% 费率
  一级传播者额外获得 20% 费率（裂变奖励）
  协议额外获得 10%

不设三级及以上，防止 MLM 结构
```

### 6.4 传播 NFT（Spreader Badge）

累计传播收益达到里程碑时，自动铸造传播徽章 NFT：

| 里程碑 | NFT 名称 | 额外特权 |
|--------|---------|---------|
| 累计传播收益 ≥ 10 DOT | Bronze Spreader | 名人堂投票权 +0.1 |
| 累计传播收益 ≥ 50 DOT | Silver Spreader | 名人堂投票权 +0.3；下届传播费率 +1% |
| 累计传播收益 ≥ 200 DOT | Gold Spreader | 名人堂投票权 +0.5；下届传播费率 +2%；颁奖典礼前排观礼席 |

> Spreader Badge 仅影响名人堂投票权，不影响 PolkaInk 主协议权重，两套系统隔离。

### 6.5 传播激励池总量控制

```
传播激励池 = 提名费的 15% + 投票费的 20% + 协议专项拨款

每届传播激励池上限 = 总奖池的 25%
（若传播激励超额，按比例折算支付，不透支其他池子）
```

## 7. 预测市场

预测市场是名人堂最重要的**纯金融层**，与选举结果挂钩但独立运营。它不影响投票结果，但为投机者提供获利渠道，同时为协议创造额外收入。

### 7.1 预测市场类型

#### 类型 A：名次预测（最高流动性）

```
标的：某候选人最终排名
赔率：由市场自动做市（AMM 机制，初始赔率由 CII 排名设定）
下注单位：DOT
最小下注：1 DOT
最大下注：2000 DOT / 地址 / 候选人
```

**赔率示例：**

| 市场预期 | 候选人 A | 候选人 B | 候选人 C |
|---------|---------|---------|---------|
| 入选概率 | 75% | 45% | 12% |
| 赔率（入选） | 1.25× | 2.0× | 7.5× |
| 赔率（未入选） | 3.8× | 1.6× | 1.1× |

#### 类型 B：排名预测（高风险高收益）

```
标的：预测某候选人最终排名（第 1 / 2 / 3 / 4-6 / 7-10 / 未入选）
收益倍数：按市场流动性和赔率实时计算
```

#### 类型 C：黑马预测（社区最爱）

```
标的：初选排名 20–30 的候选人中，预测谁能在决选翻盘进 Top 10
赔率：统一起始 5×，随下注量动态调整
吸引力：以小博大，社区参与度极高
```

#### 类型 D：候选人对战（PVP 下注）

```
标的：两个候选人 A vs B，谁的最终排名更高？
赔率：初始 1:1，随下注倾斜动态调整
特点：简单直观，适合新用户入场
最小下注：0.5 DOT
```

### 7.2 预测市场时间线

```
贡献公示期开始（W3）→ 预测市场开盘
初选投票结束（W6）  → 部分市场结算（基于初选结果）
决选投票结束（W8）  → 所有市场关闭，等待最终结果
公示争议期（W9）    → 结果冻结，不可下注
颁奖存档（W10）     → 最终结算，收益分发
```

### 7.3 预测市场收入结构

```
每笔下注：协议收取 3% 手续费（平台费）

手续费分配：
  ├── 50% → PolkaInk 主 Treasury（最重要的反哺来源）
  ├── 30% → 名人堂运营储备
  └── 20% → 传播激励池

赔注池：
  └── 100%（扣除手续费后）→ 押对结果的下注者按比例分配
```

### 7.4 预测市场流动性引导

冷启动问题：早期流动性不足导致赔率失真。解决方案：

- **协议做市**：PolkaInk Treasury 提供初始流动性（每届 500 DOT），当市场流动性超过 5000 DOT 后自动撤出并收回本金
- **做市奖励**：早期（W3–W4）提供流动性的地址，额外获得 0.5% 年化等效奖励
- **LP NFT**：提供流动性超过 100 DOT 的地址获得 Liquidity Provider 徽章，享有下届费率减免 0.5%

## 10. 资金流向与 PolkaInk 反哺

这是设计的核心：名人堂产生的每一分钱，都通过清晰的链上规则，部分回流到 PolkaInk 主协议，形成正向循环。

### 10.1 资金流总图

```
名人堂总收入
    │
    ├─────────────────────────────────────────────────────────────────
    │                                                                  │
    ▼                                                                  ▼
名人堂内部分配（65%）                              PolkaInk 主 Treasury（35%）
    │                                                                  │
    ├── 当选者支持者奖池（35%）                     ├── 主协议基础设施维护
    ├── 传播激励池（15%）                           ├── Archive Council 运营
    ├── 预测市场本金池（100% 流转后返还）           ├── PolkaInk Grant 申请资金补充
    └── 运营储备（15%）                             └── 主协议 Historian 奖励池补充
```

### 10.2 对 PolkaInk 主协议的量化反哺

| 反哺项目 | 来源 | 预计每届金额（DOT，保守/积极） |
|---------|------|--------------------------|

## 11. 防操纵与公平保障

### 11.1 已知攻击向量与对应设计

| 攻击类型 | 具体描述 | 对应防御 |
|---------|---------|---------|
| **女巫提名** | 用多个地址批量联署，制造虚假人气 | 联署地址须持有 ≥ 5 DOT 且账户存活 ≥ 30 天 |
| **刷票攻击** | 用多个地址对同一候选人大量投票 | 每票均需缴纳投票费，多地址成本线性增加 |
| **超级持鲸控票** | 单一大户用超大金额 Super Vote 控制结果 | Super Vote 每地址每届限 1 次，且权重仅 3× |
| **预测市场操纵** | 先大量押注某结果，再刷票影响选举结果改变赔率 | 下注上限 2000 DOT / 地址；投票与下注账户分析（链上可见）|
| **传播刷量** | 自己生成链接然后自己点击完成行为 | 传播奖励计算排除推广人本人的行为 |
| **虚假贡献证据** | 提交无法核实的贡献 | 社区核实机制 + 举报奖励 + 5 DOT 扣罚 |
| **晚期大量买票** | 在决选最后关头砸钱改变结果 | 决选最后 24h 内新增投票权重折半（防临时砸场）|

### 11.2 Archive Council 的名人堂角色

名人堂公示争议期（W9）由 Archive Council 负责：

- 任何成员可缴纳 **10 DOT 保证金**提出异议（异议理由须 ≥ 100 字）
- 若异议成立（Council 4/7 通过）→ 该候选人取消资格，顺位替补，提出异议者保证金返还 + 5 DOT 奖励
- 若异议不成立 → 保证金进入奖池
- 合理异议范围：候选人资质造假、贡献证据严重虚假、违反 Polkadot 社区准则

### 11.3 透明度承诺

所有以下数据实时链上公开：

- 每位候选人的实时得票数与权重
- 预测市场各标的的实时赔率与总下注量
- 传播链接追踪（推广人地址公开，但不强制披露真实身份）
- 奖池各组成部分的实时金额
- Archive Council 的所有 veto 记录与理由
